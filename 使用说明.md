## 1. 快速开始

### 基础信息

| 项目 | 说明 |
|---|---|
| 服务地址 | `https://gmail.freeopenai.me` |
| API Token | 在管理后台 `/settings` 页面设置 |
| 鉴权方式 | Query 参数 `auth_token`、Header `X-Auth-Token` 或 `Authorization: Bearer <token>` |

### 最简调用示例（OpenAI 场景）

```bash
BASE=https://gmail.freeopenai.me
TOKEN=your_api_token

# 第一步：领取一个别名邮箱
curl "$BASE/api/alias?project=openai&auth_token=$TOKEN"

# 第二步：用该别名去 OpenAI 注册/登录，触发验证码邮件

# 第三步：拉取验证码
curl "$BASE/api/code?project=openai&mail=<alias>&auth_token=$TOKEN"

# 第四步：标记别名为已使用
curl -X POST "$BASE/api/alias/mark" -d "auth_token=$TOKEN&mail=<alias>&project=openai&status=1"
```

## 2. 管理后台

访问 `https://gmail.freeopenai.me` 使用管理员密码登录（与 API Token 相同）。

### 2.1 账号管理页 `/accounts`

账号列表展示所有已授权的 Gmail 邮箱，每行右侧有以下操作按钮：

| 按钮 | 说明 |
|---|---|
| **初始化别名** | 为该账号增量生成所有点号变体别名并写入数据库。**已存在的别名不会被覆盖**（保留已有的使用状态标记）。适用于新授权账号或之前初始化失败的账号 |
| **重新授权** | 跳转到 OAuth 授权页面，重新获取该账号的 Gmail 访问权限 |
| **禁用 / 启用** | 禁用后该账号不会在别名分配和验证码接口中被使用，但保留授权凭证 |
| **删除** | 从数据库中删除账号记录（不删除 token 文件） |

### 2.2 系统设置页 `/settings`

- **API 令牌**：设置所有外部接口的访问密码
- **Token 刷新间隔**：后台自动刷新 Gmail access token 的频率（默认 6 小时）
- **立即刷新**：手动触发一次全量 token 刷新

### 2.3 初始化别名注意事项

- 点击"初始化别名"后会弹出确认对话框
- 初始化为**同步操作**，对于大账号（如 18 字符用户名，131072 个变体）需要几秒时间
- 已在数据库中的别名不会被重新写入，状态标记（已用/未用）完整保留
- 初始化结果通过页面顶部的提示消息反馈，例如：
  - `账号 xxx@gmail.com 别名初始化完成，新增 131072 个别名`
  - `账号 xxx@gmail.com 别名初始化完成，新增 0 个别名`（说明别名已全部存在）

## 3. 邮箱别名机制说明

### 3.1 点号变体原理

Gmail 对邮件地址中的点号不敏感，以下地址收件完全等价：

```
linlang156@gmail.com
l.inlang156@gmail.com
lin.lang.156@gmail.com
l.i.n.l.a.n.g.1.5.6@gmail.com
```

系统将每个 Gmail 账号的用户名所有可能的点号插入位置（`2^(n-1)` 种）预先枚举存入数据库，每次请求分配一个。

### 3.2 用户名长度与变体数量

| 用户名长度 | 理论变体数 | 实际生成数 |
|---|---|---|
| 10 字符 | 512 | 512 |
| 14 字符 | 8,192 | 8,192 |
| 18 字符 | 131,072 | 131,072 |
| 20 字符 | 524,288 | 300,000（采样上限） |

> 超过 19 字符的账号会随机采样 300,000 个变体，始终包含无点号的原始形式。

### 3.3 别名发放规则

- 每个别名在同一项目下**只发放一次**（`UNIQUE(project, alias_email)`）
- 发放后状态为"未使用（status=2）"，用户标记为"已使用（status=1）"后不会再被分配
- `openai` / `deepseek` 项目：只发放**纯点号别名**，不追加 `+suffix`
- `codeflicker` 项目：每个点号别名下最多发放 5 个 `+suffix` 变体

## 4. 接口：获取别名邮箱 `/api/alias`（GET）

### 请求参数

| 参数 | 必填 | 说明 |
|---|---|---|
| `project` | 是 | 项目名，当前支持：`openai`、`deepseek`、`codeflicker` |
| `auth_token` | 是 | API 令牌 |
| `real_email` | 否 | 限定从指定 Gmail 账号的别名池中挑选，例如 `linlang156@gmail.com` |

> 不传 `real_email` 时，系统从所有活跃且未禁用的 Gmail 账号中**随机**分配别名。

### 成功响应

```json
{
  "ok": true,
  "project": "openai",
  "alias": "lin.lang.156@gmail.com",
  "real_email": "linlang156@gmail.com",
  "plus_suffix": null,
  "max_plus_aliases": 0,
  "request_id": "abc12345"
}
```

| 字段 | 说明 |
|---|---|
| `alias` | 分配出的别名邮箱，直接拿去注册/登录 |
| `real_email` | 该别名背后的真实 Gmail 账号 |
| `plus_suffix` | `+suffix` 部分，`openai`/`deepseek` 项目始终为 `null` |

### 示例

```bash
# 随机从所有账号分配
curl "https://gmail.freeopenai.me/api/alias?project=openai&auth_token=YOUR_TOKEN"

# 指定从特定账号分配
curl "https://gmail.freeopenai.me/api/alias?project=openai&real_email=linlang156@gmail.com&auth_token=YOUR_TOKEN"
```

### 错误码

| error | HTTP | 说明 |
|---|---|---|
| `project_required` | 400 | 未传 project |
| `project_not_found` | 404 | 项目配置文件不存在 |
| `invalid_auth_token` | 401 | Token 错误 |
| `no_active_account` | 404 | 没有可用的活跃 Gmail 账号 |
| `no_available_alias` | 409 | 该项目下所有别名已发放完毕 |
| `rate_limited` | 429 | 单 IP 60 秒内超过 30 次 |

## 5. 接口：获取验证码 `/api/code`（GET）

### 请求参数

| 参数 | 必填 | 说明 |
|---|---|---|
| `project` | 是 | 项目名：`openai`、`deepseek`、`codeflicker` |
| `mail` | 是 | 收件邮箱，可以是别名或真实 Gmail，系统自动解析 |
| `auth_token` | 是 | API 令牌 |
| `time` | 否 | 时间窗口，如 `30m`、`2h`、`5h`。默认 `30m` |

### 成功响应

```json
{
  "ok": true,
  "project": "openai",
  "mail": "lin.lang.156@gmail.com",
  "time": "30m",
  "resolved_account": "linlang156@gmail.com",
  "code": "482913",
  "subject": "你的临时 ChatGPT 登录代码",
  "from": "noreply@tm.openai.com",
  "to": "lin.lang.156@gmail.com",
  "delivered_to": "lin.lang.156@gmail.com",
  "snippet": "输入此临时验证码以继续：482913",
  "internal_date": "1745318400000",
  "matched_query": "from:\"noreply@tm.openai.com\" to:\"lin.lang.156@gmail.com\" after:1745316600",
  "request_id": "abc12345"
}
```

| 字段 | 说明 |
|---|---|
| `code` | **6 位验证码** |
| `resolved_account` | 解析出的真实 Gmail 账号 |
| `matched_query` | 实际执行的 Gmail 搜索语句，便于调试 |

### 未命中（验证码未到）

```json
{
  "ok": false,
  "error": "code_not_found",
  "project": "openai",
  "mail": "lin.lang.156@gmail.com",
  "time": "30m",
  "resolved_account": "linlang156@gmail.com",
  "request_id": "abc12345"
}
```

> 推荐轮询策略：触发验证码发送后，每 **3~5 秒**调用一次，最多 **60~90 秒**。

### 示例

```bash
# 基本查询（30分钟窗口）
curl "https://gmail.freeopenai.me/api/code?project=openai&mail=lin.lang.156@gmail.com&auth_token=YOUR_TOKEN"

# 扩大时间窗口到 2 小时
curl "https://gmail.freeopenai.me/api/code?project=openai&mail=lin.lang.156@gmail.com&time=2h&auth_token=YOUR_TOKEN"
```

### 错误码

| error | HTTP | 说明 |
|---|---|---|
| `project_required` | 400 | 未传 project |
| `mail_required` | 400 | 未传 mail |
| `invalid_time` | 400 | 时间格式错误，支持 `30m`、`2h` 等 |
| `invalid_auth_token` | 401 | Token 错误 |
| `account_not_authorized` | 404 | mail 对应的真实账号未授权 |
| `account_not_active` | 409 | 账号存在但 token 失效，需重新授权 |
| `token_refresh_failed` | 409 | 刷新 Gmail access token 失败 |
| `gmail_query_failed` | 500 | 调用 Gmail API 失败 |
| `code_not_found` | 200 | 时间窗口内未找到验证码（`ok: false`） |
| `rate_limited` | 429 | 超过速率限制 |

## 6. 接口：标记别名状态 `/api/alias/mark`（POST / GET）

用于在注册成功后将别名标记为已使用，避免再次被分配。

### 请求参数

| 参数 | 必填 | 说明 |
|---|---|---|
| `auth_token` | 是 | API 令牌 |
| `mail` | 是 | 别名邮箱地址 |
| `project` | 是 | 项目名 |
| `status` | 是 | `1` = 已使用，`2` = 未使用 |

### 成功响应

```json
{
  "ok": true,
  "mail": "lin.lang.156@gmail.com",
  "project": "openai",
  "status": 1,
  "outcome": "updated",
  "request_id": "abc12345"
}
```

`outcome` 可能值：`updated`（更新已有记录）、`inserted`（新建记录）。

### 示例

```bash
# POST form 方式（推荐）
curl -X POST "https://gmail.freeopenai.me/api/alias/mark" \
  -d "auth_token=YOUR_TOKEN&mail=lin.lang.156@gmail.com&project=openai&status=1"

# GET 方式
curl "https://gmail.freeopenai.me/api/alias/mark?auth_token=YOUR_TOKEN&mail=lin.lang.156@gmail.com&project=openai&status=1"
```

## 7. 接口：账号管理

### 7.1 获取账号列表 `GET /api/accounts`

| 参数 | 说明 |
|---|---|
| `auth_token` | 必填 |
| `status` | `active`（默认）或 `all` |
| `include_disabled` | `true` / `false`，默认 `false` |

```bash
curl "https://gmail.freeopenai.me/api/accounts?auth_token=YOUR_TOKEN&status=all"
```

响应示例：

```json
{
  "ok": true,
  "count": 2,
  "accounts": [
    {
      "email": "linlang156@gmail.com",
      "status": "active",
      "disabled": false,
      "last_refreshed_at": "2026-04-22T12:24:54+00:00",
      "last_error": ""
    }
  ]
}
```

### 7.2 强制刷新 Token `POST /api/accounts/refresh`

```bash
# 刷新单个账号
curl -X POST "https://gmail.freeopenai.me/api/accounts/refresh" \
  -H "X-Auth-Token: YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"email": "linlang156@gmail.com"}'

# 全量刷新（不传 email）
curl -X POST "https://gmail.freeopenai.me/api/accounts/refresh" \
  -H "X-Auth-Token: YOUR_TOKEN" \
  -d '{}'
```

### 7.3 启用 / 禁用账号 `POST /api/accounts/toggle-disable`

```bash
curl -X POST "https://gmail.freeopenai.me/api/accounts/toggle-disable" \
  -H "X-Auth-Token: YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"email": "linlang156@gmail.com", "disabled": true}'
```

## 10. 完整使用流程示例

### Bash 脚本（OpenAI）

```bash
#!/usr/bin/env bash
set -e

BASE=https://gmail.freeopenai.me
TOKEN=your_api_token

# 1. 领取别名
ALIAS=$(curl -s "$BASE/api/alias?project=openai&auth_token=$TOKEN" | python3 -c "import json,sys; print(json.load(sys.stdin)['alias'])")
echo "Got alias: $ALIAS"

# 2. 用该别名去 OpenAI 注册或登录（触发验证码发送）
# ... 省略 ...

# 3. 轮询验证码，最多等 90 秒
CODE=""
for i in $(seq 1 18); do
  RESP=$(curl -s "$BASE/api/code?project=openai&mail=$ALIAS&auth_token=$TOKEN")
  CODE=$(echo "$RESP" | python3 -c "import json,sys; d=json.load(sys.stdin); print(d.get('code',''))")
  if [ -n "$CODE" ]; then
    echo "Got code: $CODE"
    break
  fi
  echo "Waiting ($i/18)..."
  sleep 5
done

if [ -z "$CODE" ]; then
  echo "Timeout: no code received"
  exit 1
fi

# 4. 提交验证码给 OpenAI
# ... 省略 ...

# 5. 标记为已使用
curl -s -X POST "$BASE/api/alias/mark" \
  -d "auth_token=$TOKEN&mail=$ALIAS&project=openai&status=1" > /dev/null
echo "Marked $ALIAS as used"
```

### Python 脚本

```python
import time
import requests

BASE = "https://gmail.freeopenai.me"
TOKEN = "your_api_token"


def get_alias(project: str, real_email: str = None) -> str:
    params = {"project": project, "auth_token": TOKEN}
    if real_email:
        params["real_email"] = real_email
    resp = requests.get(f"{BASE}/api/alias", params=params, timeout=15)
    data = resp.json()
    if not data.get("ok"):
        raise RuntimeError(f"get_alias failed: {data}")
    return data["alias"]


def wait_code(mail: str, project: str, timeout: int = 90, interval: float = 5.0) -> str:
    deadline = time.time() + timeout
    while time.time() < deadline:
        resp = requests.get(
            f"{BASE}/api/code",
            params={"project": project, "mail": mail, "time": "30m", "auth_token": TOKEN},
            timeout=15,
        )
        data = resp.json()
        if data.get("ok") and data.get("code"):
            return data["code"]
        time.sleep(interval)
    raise TimeoutError(f"No code received for {mail} in {timeout}s")


def mark_used(mail: str, project: str) -> None:
    requests.post(
        f"{BASE}/api/alias/mark",
        data={"auth_token": TOKEN, "mail": mail, "project": project, "status": 1},
        timeout=15,
    )


if __name__ == "__main__":
    alias = get_alias("openai")
    print("Alias:", alias)

    # 用 alias 去 OpenAI 登录/注册，触发验证码...

    code = wait_code(alias, "openai")
    print("Code:", code)

    # 提交验证码...

    mark_used(alias, "openai")
    print("Done.")
```

## 12. 常见问题

**Q: 调用 `/api/alias` 返回 `no_available_alias`？**

A: 该项目下所有别名已被发放过。可以：
- 在管理后台 `/accounts` 点击"初始化别名"为账号补充别名（已有的不会被覆盖）
- 添加新的 Gmail 账号授权
- 使用 `real_email` 参数指定未耗尽的账号

**Q: 调用 `/api/code` 返回 `account_not_authorized`？**

A: 传入的邮箱（含别名）对应的真实 Gmail 账号未完成 OAuth 授权。需要在管理后台 `/oauth` 页面为该账号授权。

**Q: 调用 `/api/code` 返回 `code_not_found`？**

A: 时间窗口内没有找到验证码邮件。建议：
1. 确认验证码邮件已发送到该别名
2. 适当放宽时间窗口：`&time=2h`
3. 多等几秒再重试（邮件延迟）

**Q: 点击"初始化别名"提示"新增 0 个别名"？**

A: 该账号的所有别名已全部存在于数据库中，无需重新初始化，可以正常使用。

**Q: 已用过的别名会被再次分配吗？**

A: 不会。调用过 `/api/alias` 的别名会在该项目下记录为已发放，再次请求会分配不同的别名。调用 `/api/alias/mark?status=1` 标记为"已使用"后，状态会被保留，不影响其他未发放的别名。

**Q: 一个别名能用于多个项目吗？**

A: 可以。别名的发放记录是**按项目隔离**的，同一别名可以在 `openai` 和 `deepseek` 项目下各独立发放一次。

**Q: 账号状态显示"需重新授权"怎么办？**

A: 点击该账号右侧的"重新授权"按钮，按照 OAuth 流程重新完成 Google 授权即可。Token 过期通常因 Google OAuth 应用处于"测试"模式（7 天过期），建议在 Google Cloud Console 将应用发布为"生产"状态。
